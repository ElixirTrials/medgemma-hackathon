# Plan: Improve Modify Modal for Entity & Criterion Review

## Context

Reviewers need to see what the system originally selected (SNOMED codes, UMLS CUIs, entity-relation-value-unit mappings) so they can confirm or correct it. Currently, clicking "Modify" shows inline edit fields without clearly presenting the original system selections. We'll convert both EntityCard and CriterionCard modify flows to proper Radix Dialog modals with a clear "System Selected / Your Corrections" layout. The criterion text should NOT be freely editable (it's part of the audit trail); instead, reviewers adjust the text boundary using a highlight-and-drag UI over the source context.

---

## Files to Create

### 1. `apps/hitl-ui/src/components/EntityModifyDialog.tsx`
Radix Dialog modal for editing entity terminology codes. Uses `max-w-lg`.

**Layout (top to bottom):**
- **Title bar:** "Modify Entity: {entity.text}" + EntityTypeBadge + confidence badge
- **"System Selected" panel** (read-only, `bg-muted/30 border rounded-lg p-4`):
  - All current terminology codes as `TerminologyBadge` components
  - Preferred term, grounding method, confidence
- **"Your Corrections" section:**
  - One `TerminologyCombobox` per relevant system (existing `getRelevantSystems()` logic)
  - Change indicator `(changed)` next to any field differing from original
  - Manual CUI/SNOMED fallback inputs
- **Footer:** Save / Cancel buttons

**Callback contract:** The dialog receives `onSave: (payload: Omit<EntityActionRequest, 'action' | 'reviewer_id'>) => void`. The dialog is only responsible for "what changed." The parent EntityCard wraps it as `onAction(entity.id, { action: 'modify', reviewer_id: 'current-user', ...payload })`. The dialog never needs the entity ID.

**State reset:** When `open` transitions to `false` (via `onOpenChange`), reset all local edit state to the current entity values.

### 2. `apps/hitl-ui/src/components/CriterionModifyDialog.tsx`
Radix Dialog modal for editing criterion structured fields and adjusting text boundaries. Uses `max-w-2xl`.

**Layout:**
- **Title bar:** "Modify Criterion" + type/category/confidence badges (imported from CriterionCard exports)
- **"System Selected" panel** (read-only):
  - Current field mappings via read-only `FieldMappingBadges` (no `onEditClick`)
  - Temporal constraint and numeric threshold badges
  - Entities with their terminology badges
- **Text Boundary Adjuster** (see component #3 below)
- **Structured Fields section:** embedded `StructuredFieldEditor` with `initialValues` passed in from parent via `structuredInitialValues` prop (built by `buildInitialValues(criterion)` in CriterionCard)
- **Footer:** Save / Cancel

**Callback contract:** The dialog receives `onAction: (action: ReviewActionRequest) => void`. The parent CriterionCard wraps it as `(action) => onAction(criterion.id, action)`. The dialog never needs the criterion ID.

**State reset:** Same pattern as EntityModifyDialog — reset on close.

### 3. `apps/hitl-ui/src/components/TextBoundaryAdjuster.tsx`
A new component for adjusting the criterion text boundaries within the source context.

**How it works:**
- Uses `pdfjs` (already loaded via `react-pdf`) to extract the full text content from the criterion's `page_number`
- Locates the criterion `text` within the page text to determine surrounding context
- Displays the page text with the selected portion highlighted (yellow background)
- Draggable handles at the start and end of the highlight let the reviewer expand or shrink the selection word-by-word
- Below the text display, shows the current selected text as a preview

**Implementation details:**
- `pdfjs.getDocument(pdfUrl)` → `doc.getPage(pageNumber)` → `page.getTextContent()` → concatenate text items
- Find criterion text position within page text using substring search
- Track `startWordIndex` and `endWordIndex` into a word array
- Drag handles implemented with `onMouseDown` + `onMouseMove` listeners, snapping to word boundaries
- CSS `user-select: none` during drag to prevent text selection interference
- Fallback: if page text extraction fails or criterion text not found, show the text as read-only (no adjustment possible)

Props: `{ criterionText, pageNumber, pdfUrl, onChange: (newText: string) => void }`

**Visual design:**
```
┌──────────────────────────────────────────────┐
│ Source context (page 5):                     │
│                                              │
│ ...prior therapy. [▍Patients must have       │
│ documented HbA1c ≥ 7.5% within 3 months▍]   │
│ of screening visit. Additional...            │
│                                              │
│ ◄──  drag handles at edges  ──►              │
│                                              │
│ Selected: "Patients must have documented     │
│           HbA1c ≥ 7.5% within 3 months"     │
└──────────────────────────────────────────────┘
```
